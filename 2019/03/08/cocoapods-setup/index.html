三步安装完成Cocoapods

Posted on 2019-03-08 | Edited on 2019-04-13 | In cocoapods

前言：公司给了一台新MAC，需要半天内搞完全部的配置环境，然后就先安装了一下pod，花了一小时左右安装完成，安装完成pod之后才安装的Xcode。

第一步：sudo gem update –system

打开终端输入

sudo gem update –system

终端提示
Password: （输入Mac的用户密码）

安装成功提示

Latest version already installed. Done

注意事项

1、gem update --system命令前没有加 sudo

将提示以下错误内容

Updating rubygems-update
Fetching: rubygems-update-3.0.3.gem (100%)
ERROR:  While executing gem ... (Gem::FilePermissionError)
You don't have write permissions for the /Library/Ruby/Gems/2.3.0 directory.

2、出现错误描述 ERROR: While executing...，具体内容如下

Password:
Updating rubygems-update
Fetching: rubygems-update-3.0.3.gem (100%)
Successfully installed rubygems-update-3.0.3
。。。中间过程省略
ERROR:  While executing gem ... (Errno::EPERM)
Operation not permitted @ rb_sysopen - /System/Library/Frameworks/Ruby.framework/Versions/2.3/usr/bin/gem

问题原因为网络连接异常，重新执行命令

第二步：sudo gem install cocoapods -n /usr/local/bin

安装成功提示

Successfully installed xcodeproj-1.8.2
。。。中间过程省略
Done installing documentation for xcodeproj, molinillo, cocoapods-try, netrc, cocoapods-trunk, cocoapods-stats, cocoapods-search, cocoapods-plugins, cocoapods-downloader, cocoapods-deintegrate, fuzzy_match, cocoapods-core, cocoapods after 11 seconds
13 gems installed

注意事项

1、查看gems当前来源gem sources -l

提示如下：

*** CURRENT SOURCES ***

https://rubygems.org/

如果能访问https://rubygems.org/，无需更换gems托管服务地址

2、执行 sudo gem install cocoapods,提示如下内容

Fetching ruby-macho-1.4.0.gem
。。。中间过程省略
Successfully installed nanaimo-0.2.6
ERROR:  While executing gem ... (Gem::FilePermissionError)
You don't have write permissions for the /usr/bin directory.

问题原因是苹果系统升级 OSX EL Capitan之后/usr/bin没有写入的权限，在命令后面加上 -n /usr/local/bin

第三步：pod setup

运行命令会提示安装git，必须安装git。此过程耗时1个小时左右，网速好的话，可以快一些。
